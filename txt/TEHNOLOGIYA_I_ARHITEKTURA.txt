ГЛАВА 2. РАЗРАБОТКА И ТЕСТИРОВАНИЕ ПРИЛОЖЕНИЯ
2.1. Технология и архитектура
Для разработки мобильного приложения «RestaurantBooking» был выбран современный стек технологий, обеспечивающий высокую производительность и стабильность работы. Основным языком программирования был выбран Kotlin — официально рекомендуемый язык для разработки под Android. В качестве среды разработки (IDE) использовалась Android Studio, а для построения пользовательского интерфейса применялась классическая система макетирования на основе XML и библиотеки Material Components, что обеспечило широкую совместимость и гибкость настройки визуальных элементов.

Конкурентные преимущества Kotlin:
1) Null Safety — встроенная защита от обращений к пустым ссылкам (null) на этапе компиляции, что критически важно для предотвращения падений приложения при работе с базой данных ресторанов и броней.
2) Краткость кода — использование лаконичного синтаксиса позволило сократить объем кодовой базы по сравнению с Java, повысив её читаемость и облегчив поддержку логики бронирования.
3) Полная совместимость с Java — это позволило бесшовно интегрировать сторонние библиотеки и использовать стандартные возможности Android SDK.
4) Корутины (Coroutines) — мощный инструмент для асинхронного программирования. С их помощью реализовано взаимодействие с базой данных Room без блокировки пользовательского интерфейса.
5) Расширения (Extension Functions) — позволили расширить функционал стандартных классов Android без изменения их исходного кода, что сделало код приложения более чистым.
6) Data-классы — обеспечили быструю и безошибочную реализацию моделей данных (User, Restaurant, Review, Booking), автоматически генерируя необходимые методы сравнения и копирования.

Дополнительные технологии и инструменты:
1) Room Persistence Library — объектно-ориентированная надстройка над SQLite, использованная для надежного локального хранения данных о ресторанах, пользователях и совершаемых бронях.
2) Architecture Components (ViewModel, LiveData) — использование этих компонентов позволило эффективно управлять жизненным циклом данных и синхронизировать состояние интерфейса с бизнес-логикой.
3) Gradle — гибкая система сборки, обеспечивающая управление зависимостями и автоматизацию процесса компиляции приложения.

Инструментальные возможности Android Studio:
1) Intelligent Code Editor — продвинутые инструменты автодополнения и анализа кода, ускоряющие написание логики приложения.
2) Layout Editor — визуальный редактор, позволивший детально проработать макеты экранов бронирования и админ-панели с использованием ConstraintLayout.
3) Android Emulator — средство для тестирования работы приложения на различных конфигурациях устройств, включая проверку адаптивности верстки под разные размеры экранов.
4) Profiling Tools — инструменты для мониторинга потребления памяти и ресурсов процессора, что позволило оптимизировать работу длинных списков ресторанов.
